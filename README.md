# Minimal-Siyu

> B 站「思宇频道」视频讲解素材沉淀仓库。
>
> 把每一篇看过的论文、读过的源码、推过的公式，沉淀成一段可以讲清楚的内容。

---

## 仓库定位

这里**不是**个人学习计划，**是**视频内容的「编辑部」：

- 一期视频 = `episodes/` 下的一个目录
- 一个目录里同时放讲稿、关键图、最小代码片段
- 视频发布后，回填 B 站链接，目录本身就成为该期视频的「show notes」

---

## 目录结构

```text
.
├── episodes/                       视频期次（一期一目录）
│   ├── _template/                  单期内容模板（新建期次时复制一份）
│   │   └── notes.md
│   ├── 2026-W22-NeRF-3DGS-VGGT-IR/ 第一期：NeRF/3DGS/VGGT 与逆渲染（领域全景）
│   │   ├── notes.md
│   │   ├── figures/
│   │   └── code/
│   └── 2026-W23-vLLM-LoRA-SFT/     第二期：vLLM × LoRA × SFT（本地部署三件套）
│       ├── notes.md
│       ├── figures/
│       └── code/
├── Basic/                          基础内容选题地图（episodes 的素材池）
│   └── README.md                   编程 / 常用库 / 数学 三大类的可讲主题
├── logo.jpg

```

**命名规则**：`YYYY-WNN-主题`（年 + ISO 周数 + 主题短名）。

## 单期内容结构

每一期 `notes.md` 都按同一份骨架展开，方便后期剪辑、回看，也方便观众跳读：

1. **背景 & 动机**：要解决什么问题，为什么值得讲
2. **核心方法**：一句话总结 + 系统流程图
3. **关键公式**：2–3 个最核心的式子，配口语化解释
4. **实验亮点**：最能体现方法价值的 1–2 张图 / 表
5. **局限**：作者自承 + 个人补充
6. **5 分钟讲稿**：「如果只能讲一件事」的口播版
7. **前置 / 延伸阅读**：3 篇相关工作做引子或后续

## 新建一期的工作流

1. 复制模板：`cp -r episodes/_template episodes/YYYY-WNN-主题`
2. 填 `notes.md`：按七段骨架写满，先粗后细
3. 抽图存证：把要在视频里出现的截图 / 示意图放到 `figures/`
4. 写最小代码：能跑、能讲清楚一个点即可，存到 `code/`
5. 录制 → 上传 → 回填本 README 的「期次列表」
